# AI 工作文档

> 本文档用于规范人机协作流程，提升 AI 辅助游戏设计与开发的效率和质量。

---

## 1. 工作空间定位

- **路径**：`C:\Users\huang\ai-workspace`
- **用途**：存放提示词模板、AI 输出产物、迭代记录、参考资料等人机协作相关文件。

---

## 2. 协作原则

| 原则 | 说明 |
|------|------|
| **先做再问** | 当任务是“做游戏/做功能”时，先动手构建可运行版本，再迭代优化。 |
| **小步快跑** | 一次只做一个功能点，快速验证，避免一次性改动过多。 |
| **文档为辅** | 设计文档是起点，不是终点；优先通过代码/原型推进。 |
| **上下文优先** | 尽量把当前任务、项目目标、约束条件一次性说清楚。 |
| **中文优先** | 所有自然语言沟通使用中文，代码/配置保持英文。 |

## 4. AI 协作工作流

### 4.1 需求输入

每次请求尽量包含以下信息：

1. **目标**：要做什么？
2. **背景**：为什么做？当前做到哪一步？
3. **约束**：技术栈、性能要求、风格限制等。
4. **验收标准**：做成什么样算完成？

### 4.2 输出规范

- 代码必须可运行，优先给出完整文件或 patch。
- 复杂改动给出简要说明和验证步骤。
- 遇到歧义，先给出最可能正确的方案并说明假设。

### 4.3 迭代闭环

```
需求 → 原型/代码 → 运行验证 → 反馈 → 修改 → 验收
```

- 不将 API 密钥、账号密码等敏感信息写入代码或文档。
